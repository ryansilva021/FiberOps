Visão Geral da Arquitetura de Simulação

Como montar um lab virtual idêntico a uma rede ISP real

Uma rede FTTH real de provedor tem: OLT (equipamento central que gerencia as fibras), ONUs/ONTs (equipamentos na casa do cliente) e um ACS para gerenciamento TR-069. Em Docker, replicamos toda essa pilha com containers especializados conectados numa rede virtual isolada — sem precisar de hardware físico.

Criando a Rede Docker para o Lab ISP

Subnet isolada que simula a rede interna do provedor

Toda a comunicação entre OLT, ONUs, ACS e FiberOps acontece dentro de uma rede Docker dedicada com IPs fixos. Isso garante que os containers se encontrem pelos mesmos endereços sempre que o lab subir.

Criar a rede manualmente (opção 1)

# Cria a rede isolada para o lab ISP
docker network create \
  --driver bridge \
  --subnet 172.20.0.0/24 \
  --gateway 172.20.0.1 \
  --opt com.docker.network.bridge.name=br-isp-lab \
  rede-isp

# Verificar
docker network inspect rede-isp | grep -E '"Subnet"|"Gateway"'

Via docker-compose.lab.yml (opção 2 — recomendada)

networks:
  rede-isp:
    driver: bridge
    ipam:
      config:
        - subnet: 172.20.0.0/24
          gateway: 172.20.0.1

Verificar conectividade entre containers

# Teste de ping entre FiberOps e pOLT
docker run --rm --network rede-isp alpine ping -c 3 172.20.0.10

# Ver todos os containers na rede
docker network inspect rede-isp --format '{{range .Containers}}{{.Name}} → {{.IPv4Address}}{{"\n"}}{{end}}'
⚠ Atenção: Conflito de subnet

Se a subnet 172.20.0.0/24 já for usada em outra rede Docker, mude para 192.168.100.0/24 ou qualquer /24 disponível. Use docker network ls && docker network inspect <nome> para verificar.

pOLT — OLT Programável (Broadband Forum OB-BAA)

Container que simula uma OLT GPON com API REST para provisão

A pOLT (programmable OLT) é parte do projeto open-source OB-BAA (OpenBroadband Access Abstraction) da Broadband Forum. Ela expõe a API REST POST /polt/polt_actions que o FiberOps usa para provisionar e remover ONUs — exatamente como uma OLT Huawei real, mas sem hardware.

Subir a pOLT com Docker

docker run -d \
  --name polt-sim \
  --network rede-isp \
  --ip 172.20.0.10 \
  -p 3002:3002 \
  -p 7547:7547 \
  -e POLT_NAME="OLT-LAB-01" \
  -e POLT_GPON_PORTS=4 \
  -e POLT_ONUS_PER_PORT=8 \
  broadbandforum/obbaa-polt:latest

# Verificar se subiu
curl -s http://localhost:3002/polt/status | python3 -m json.tool

ONU Simulada — Simulador de CPE GPON

Container que imita o comportamento de uma ONU na ponta do cliente

O simulador de ONU (obbaa-onu-sim) recebe comandos OMCI simulados da pOLT e responde como uma ONU real faria. Ele também pode expor uma interface TR-069 para o GenieACS gerenciar como um CPE.

Subir uma ONU simulada

docker run -d \
  --name onu-sim-01 \
  --network rede-isp \
  --ip 172.20.0.20 \
  -p 50000:50000 \
  -e ONU_SERIAL="HWTCAABBCCDD" \
  -e ONU_VENDOR_ID="HWTC" \
  -e ONU_POLT_IP="172.20.0.10" \
  -e ONU_POLT_PORT="3002" \
  -e ONU_SIM_PORT="50000" \
  -e ONU_RX_POWER="-21.5" \
  -e ONU_TX_POWER="2.1" \
  broadbandforum/obbaa-onu-sim:latest

Subir múltiplas ONUs (script bash)

#!/bin/bash
# Sobe 5 ONUs simuladas com seriais sequenciais

SERIALS=("HWTCAABB0001" "HWTCAABB0002" "HWTCAABB0003" "HWTCAABB0004" "HWTCAABB0005")
BASE_IP="172.20.0"
BASE_PORT=50000

for i in "${!SERIALS[@]}"; do
  ONU_NUM=$((i + 1))
  IP="${BASE_IP}.$((20 + i))"
  PORT=$((BASE_PORT + i))
  SERIAL="${SERIALS[$i]}"

  docker run -d \
    --name "onu-sim-0${ONU_NUM}" \
    --network rede-isp \
    --ip "$IP" \
    -p "${PORT}:${PORT}" \
    -e ONU_SERIAL="$SERIAL" \
    -e ONU_VENDOR_ID="HWTC" \
    -e ONU_POLT_IP="172.20.0.10" \
    -e ONU_SIM_PORT="$PORT" \
    -e ONU_RX_POWER="-$((18 + i * 2)).${RANDOM:0:1}" \
    broadbandforum/obbaa-onu-sim:latest

  echo "✓ ONU $ONU_NUM | IP: $IP | Serial: $SERIAL | Port: $PORT"
done

echo "
5 ONUs simuladas ativas."

Simular ONU oscilando (offline/online)

#!/bin/bash
# Simula uma ONU oscilando a cada 30 segundos
# Útil para testar os alertas NOC do FiberOps

ONU_CONTAINER="onu-sim-01"

echo "Iniciando simulação de oscilação para $ONU_CONTAINER..."
while true; do
  echo "$(date '+%H:%M:%S') → ONU OFFLINE (docker pause)"
  docker pause $ONU_CONTAINER

  sleep 15

  echo "$(date '+%H:%M:%S') → ONU ONLINE (docker unpause)"
  docker unpause $ONU_CONTAINER

  sleep 30
done

Teste de carga — 20 ONUs simultâneas

#!/bin/bash
# Provisiona 20 ONUs sequencialmente e mede o tempo

START=$(date +%s)

for i in $(seq 1 20); do
  SERIAL=$(printf "HWTCLOAD%04d" $i)
  VENDOR_SPECIFIC=$(printf "LOAD%04d" $i)

  curl -s -X POST http://localhost:3002/polt/polt_actions \
    -H "Content-Type: application/json" \
    -d "{\"requests\":[{
      \"channel_term\": $(( (i-1) / 8 + 1 )),
      \"action\": \"ADDONU\",
      \"onu_id\": $i,
      \"serial_vendor_id\": \"HWTC\",
      \"serial_vendor_specific\": \"$VENDOR_SPECIFIC\",
      \"flags\": \"present+in_o5\",
      \"management_state\": \"relying-on-vomci\"
    }]}" > /dev/null

  echo "Provisionada ONU $i/$i → serial: $SERIAL"
done

END=$(date +%s)
echo "
✓ 20 ONUs provisionadas em $((END - START)) segundos"

// src/lib/genieacs-client.js

export async function getDeviceBySerial(serial) {
  const GENIEACS_NBI = process.env.GENIEACS_NBI_URL ?? 'http://localhost:7557'
  const query = encodeURIComponent(JSON.stringify({
    "DeviceID.SerialNumber": serial
  }))
  const res = await fetch(`${GENIEACS_NBI}/devices?query=${query}`)
  if (!res.ok) throw new Error(`GenieACS NBI error: ${res.status}`)
  return res.json()
}

// src/lib/snmp-poller.js
import snmp from 'net-snmp'

export async function pollOltMetrics(ip, oids) {
  return new Promise((resolve, reject) => {
    const session = snmp.createSession(ip, 'public')
    session.get(oids, (err, varbinds) => {
      session.close()
      if (err) return reject(err)
      resolve(varbinds.map(v => ({
        oid: v.oid,
        value: v.value,
      })))
    })
  })
}

// src/lib/signal-monitor.js
// Exemplo de como processar métricas de sinal

const THRESHOLDS = {
  ok:       { min: -20,  level: 'info'    },
  alert:    { min: -25,  level: 'warn'    },
  critical: { min: -28,  level: 'error'   },
  offline:  { min: -999, level: 'error'   },
}

export function classifySignal(rxPowerDbm) {
  if (rxPowerDbm >= -20) return { status: 'ok',       ...THRESHOLDS.ok       }
  if (rxPowerDbm >= -25) return { status: 'alert',    ...THRESHOLDS.alert    }
  if (rxPowerDbm >= -28) return { status: 'critical', ...THRESHOLDS.critical }
  return { status: 'offline', ...THRESHOLDS.offline }
}

export function shouldAlert(rxPower, previousRx) {
  const current  = classifySignal(rxPower)
  const previous = classifySignal(previousRx)
  return current.status !== previous.status
}

Glossário FTTH e Referências

Terminologia GPON / FTTH

OLT (Optical Line Terminal)
Equipamento central da rede FTTH, instalado no NOC/Data Center do provedor. Concentra múltiplas PONs.
ONU / ONT (Optical Network Unit/Terminal)
Equipamento instalado na residência do cliente. Converte sinal óptico em sinal elétrico (Ethernet, WiFi, etc.).
PON (Passive Optical Network)
Rede óptica passiva — usa splitters ópticos para dividir o sinal entre múltiplos clientes sem equipamentos ativos.
GPON (Gigabit PON)
Padrão ITU-T G.984. Download 2.5 Gbps, upload 1.25 Gbps. Até 128 ONUs por PON port. Mais comum no Brasil.
XGS-PON (10G-PON)
Padrão ITU-T G.9807. Download e upload simétricos de 10 Gbps. Evolução natural do GPON.
Splitter óptico
Dispositivo passivo que divide o sinal óptico (ex: 1:8, 1:16, 1:32). Cada divisão implica perda de ~3.5 dB.
Channel Termination (channel_term)
Identificador da porta PON na OLT. Cada porta PON conecta a um tronco de fibra que vai para o splitter.
ONU ID (onu_id)
Identificador único da ONU dentro de um channel_term (0–127). Atribuído durante o processo de ranging.
Serial Number (FSID)
Identificador único físico da ONU. Composto por serial_vendor_id (4 chars) + serial_vendor_specific (8 chars hex).
RxPower (dBm)
Potência do sinal óptico recebido. Faixa típica GPON: -8 a -28 dBm. Abaixo de -28 dBm = sem sinal.

Terminologia do Lab Docker

pOLT (programmable OLT)
OLT programável do projeto OB-BAA da Broadband Forum. Suporte a YANG models e API REST para automação.
ONU Simulator (obbaa-onu-sim)
Simulador de ONU GPON em Docker. Responde ao ranging da pOLT como se fosse uma ONU física real.
OB-BAA (OLT Broadband Access Abstraction)
Framework open-source da Broadband Forum para abstração de OLTs heterogêneas. Base do pOLT simulator.
vOMCI
Virtualized OMCI — protocolo de gerência de ONU virtualizado. management_state: "relying-on-vomci" indica uso do vOMCI.
OMCI (ONU Management and Control Interface)
Protocolo G.988 para gerência de ONU através da fibra óptica. Configura serviços, VLANs, QoS na ONU.
vANI (virtual ANI)
Interface de rede virtual na OLT que representa uma ONU. Configurada com YANG models para cada cliente.
LOID (Line ID)
Identificador de linha do cliente. Usado para autenticação via LOID+LOID Password no OMCI.
GenieACS
Auto Configuration Server (ACS) open-source para TR-069/CWMP. Gerencia roteadores e CPEs dos clientes.
TR-069 / CWMP
CPE WAN Management Protocol. Protocolo para gerência remota de equipamentos de cliente (roteadores, modems).
NOC (Network Operations Center)
Centro de Operações de Rede. No FiberOps, é a tela de monitoramento em tempo real com SSE feed.
